# Contributing to resume-optimizer

Contributions are generated by an LLM, not written by hand. You provide the evidence; the LLM produces the contribution in the correct format.

---

## How to contribute (60 seconds)

1. Load **both** `resume-optimizer/SKILL.md` and this file into your LLM conversation
2. Give the LLM your evidence (see "What to provide" below)
3. Use the prompt template below to generate the contribution
4. Review the output for sanitization and accuracy
5. Fork → branch → paste the output into SKILL.md → open a PR

---

## Prompt template

Paste this into your LLM conversation after loading SKILL.md and CONTRIBUTING.md:

```
I want to contribute to this skill. Here is my evidence:

[Paste your evidence here — see "What to provide" below]

Using the skill's existing format, generate a contribution I can add to SKILL.md. Follow the format rules in CONTRIBUTING.md. Apply the sanitization checklist before outputting anything. Output only the text to insert into SKILL.md — nothing else.
```

---

## What to provide as evidence

**For a new pattern:**
- A description of the technique (what the resume did structurally)
- The outcome that validates it (callback, offer, interview — be specific about the result, not the employer)
- The function and seniority level of the candidate
- Whether you observed this in 1 resume or multiple

**For a new anti-pattern:**
- A description of the mistake (what the resume did wrong)
- Evidence that it hurts (recruiter feedback, disqualification, or observable pattern across multiple resumes)
- The function/level where it most commonly appears
- The fix

**For a function scaffold fill-in:**
- The function family (must match one of the existing scaffolds in the skill)
- The specific tactical patterns you've observed work for that function
- The outcome evidence for each pattern
- The domain vocabulary list for that function

**What you do NOT need to provide:**
- Formatted SKILL.md text — the LLM generates that
- Specific resume content — describe patterns structurally, don't quote real resumes
- Examples — the skill uses structure, formulae, and keyword lists only

---

## What the LLM will generate

The LLM will produce text in the skill's existing format:

**For patterns:**
```
### Pattern N: [Name] [CONFIDENCE-TAG]

**[Why this works / Detection criteria]**

**Formula:** `[structural template]`

**Target:** [target spec]

**Best for:** [when to apply]

**Skip when:** [when not to apply]

**Honesty check:** [one-line honesty guardrail]
```

**For anti-patterns:**
```
### Anti-Pattern N: [Name] [SCOPE-TAG]

**Detection patterns (red flags):**
- [red flag 1]
- [red flag 2]

**Why it hurts:**
- [reason 1]

**Fix:** [what to do instead]
```

Confidence tags: `[CONFIRMED]` (2+ validated sources) · `[OBSERVED]` (single source) · `[CONTEXT-DEPENDENT]` · `[PEER-REVIEW-SOURCED]` · `[METHODOLOGY]`

---

## Sanitization — the LLM runs this before outputting

The LLM should apply the following checklist to its output before returning it. If you are the LLM generating a contribution, verify each item is clear before outputting:

- [ ] No candidate names, contact info, or social handles
- [ ] No specific employer names — use tier descriptors from Appendix A.12 in SKILL.md ("BigTech-tier employer," "Series B startup," "enterprise SaaS vendor")
- [ ] No specific client names — use function descriptors from Appendix A.12 in SKILL.md ("regional bank client," "state agency engagement," "Fortune-tier retailer")
- [ ] No specific patent numbers, paper titles, or citation counts that uniquely identify a person
- [ ] No internal project codenames
- [ ] No career trajectory details that fingerprint a specific person (company sequence, tenure length at a specific employer, school name + year + company sequence)
- [ ] No dollar figures, percentages, or headcounts from a specific real resume — use structural placeholders (`$[X]M`, `[N]%`, `[N]-person`)
- [ ] No before/after quoted resume text — describe patterns structurally

---

## What won't be accepted

- Patterns based on theory or advice rather than observed outcomes
- Contributions that include identifiable information
- Advice suggesting candidates misrepresent titles, fabricate credentials, or obscure material facts
- Examples — quoted resume text in any form

---

## Git mechanics

```bash
git fork
git checkout -b pattern/[name]     # or antipattern/ or scaffold/
# paste LLM output into resume-optimizer/SKILL.md at the correct location
git add resume-optimizer/SKILL.md
git commit -m "Add [Pattern/Anti-Pattern N]: [Name]"
git push
# open PR
```

Include in the PR description: what you added, what evidence supports it, and confirmation the sanitization checklist was completed by the LLM.
